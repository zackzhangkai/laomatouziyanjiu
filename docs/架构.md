基于 Cloudflare + D1 的全栈博客系统开发
🛠 项目概述
你需要开发一个轻量级的个人博客系统，完全部署在 Cloudflare 生态中。

前端与文章： 采用静态/服务端渲染（SSR）框架，文章以 .md (Markdown) 格式存储在项目源码中。

数据库： 评论、用户数据存储在 Cloudflare D1（基于 SQLite 的边缘数据库）中。

权限机制：

游客：仅能浏览文章。

注册用户：可以发表评论、留言。

管理员（你）：在前端页面拥有专属的登录入口，登录后可以直接在页面上撰写、发布 Markdown 文章。

🎯 核心技术栈建议
框架： Astro（推荐，对 Markdown 支持极佳且原生适配 Cloudflare Pages）或 Next.js。

部署平台： Cloudflare Pages (Full-stack mode)。
部署平台： Cloudflare Pages (全栈模式)。

数据库： Cloudflare D1 (SQLite)。

身份认证： 基于 JWT (JSON Web Tokens) 的轻量级认证，或使用开源的 Auth.js / Lucide Auth，数据持久化在 D1。

第二步：路由与权限控制 (Backend API / Workers)
智能体需要实现以下全栈接口（或 Cloudflare Workers 路由）：

POST /api/auth/register：用户注册（默认角色为 'user'）。

POST /api/auth/login：用户登录，验证通过后发放 JWT，存入 HttpOnly Cookie。

GET /api/comments?article_id=xxx：公开接口，任何人可读取当前文章的评论。

POST /api/comments：需登录权限。检查 Cookie 中的 JWT，提取 user_id，向 D1 写入评论。

POST /api/articles：需管理员权限。专门用于接收页面端提交的 Markdown 文章。

第三步：解决“页面直接发文章”的核心逻辑
因为文章是用 Markdown 文件存储在仓库中的，当管理员在页面点击“发布”时，后端需要把文本变成文件。请让智能体按以下逻辑实现：

管理员在前端 /admin/new-post 页面（或主页的隐藏入口）通过 Markdown 编辑器写好文章。

点击发布，调用 POST /api/articles。

关键实现方式： 接口在验证管理员身份后，利用 GitHub API，将这段 Markdown 文本作为新文件（例如 src/content/blog/my-new-post.md）直接 Commit 并 Push 到你的博客 GitHub 仓库。

自动化触发： GitHub 收到新的 Commit 后，会自动触发 Cloudflare Pages 的 Deploy Hooks，系统自动重新构建，新文章立刻呈现在全网。

第四步：前端页面开发
首页 & 文章页： 解析项目内的 .md 文件，渲染成 HTML。

评论区组件：

如果用户未登录，显示：“请[登录]或[注册]后参与评论”。

如果用户已登录，显示评论输入框和“发表评论”按钮。

管理员专属入口：

在页面某处（或特定路由下）提供登录框。

管理员登录后，页面显示“➕ 发布新文章”的按钮，点击跳转到在线 Markdown 编辑器（支持标题、Slug、正文输入）。
